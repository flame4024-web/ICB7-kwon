# dbt MCP 전체 기능 가이드 (초보자용)

> **dbt(Data Build Tool)란?**
> 데이터 분석가나 엔지니어가 SQL을 이용해 데이터를 정제하고, 가공하고, 원하는 형태로 만들어주는 도구입니다.
> 쉽게 말하면, "원본 데이터 → 깔끔한 분석용 데이터"로 변환하는 파이프라인을 SQL 파일로 관리하는 방식입니다.
>
> **dbt MCP란?**
> 안티그래비티 AI가 이 dbt 도구를 직접 제어할 수 있도록 연결된 통로입니다.
> 말로 명령하면 AI가 dbt를 실행해 줍니다.

---

## 📌 기본 개념: dbt에서 자주 쓰는 용어

| 용어 | 설명 |
| :--- | :--- |
| **모델(Model)** | dbt에서 데이터를 변환하는 SQL 파일 하나하나를 말합니다. 실행되면 데이터베이스에 테이블이나 뷰로 만들어집니다. |
| **소스(Source)** | dbt가 원본으로 읽어오는 외부 데이터(예: 원시 데이터베이스 테이블)입니다. |
| **테스트(Test)** | 데이터에 오류가 없는지 검증하는 규칙입니다. 예: "이 컬럼에 중복값이 없어야 한다", "NULL이 없어야 한다" |
| **리니지(Lineage)** | 데이터의 흐름도입니다. 어떤 원본 데이터가 어떤 모델을 거쳐 최종 결과물로 만들어지는지 나타냅니다. |
| **컴파일(Compile)** | dbt 코드(Jinja 템플릿 포함)를 실제 실행 가능한 SQL 쿼리로 변환하는 과정입니다. |
| **빌드(Build)** | 모델 실행 + 테스트 + 시드 데이터 적재를 한 번에 순서대로 처리하는 작업입니다. |

---

## 🗂️ 기능 1: 프로젝트 조회 및 정보 확인

### ① 모델/소스 전체 목록 조회 (`dbt list`)

- **무엇을 하나요?**
  현재 dbt 프로젝트에 어떤 모델, 소스, 테스트, 시드 등이 있는지 전체 목록을 불러옵니다.
  처음 프로젝트를 파악하거나 원하는 모델을 찾을 때 사용합니다.

- **언제 쓰나요?**
  - 프로젝트에 어떤 데이터 파이프라인이 있는지 전체 구조를 보고 싶을 때
  - 특정 이름을 포함한 모델만 필터링해서 확인하고 싶을 때
  - 어떤 모델이 특정 태그로 묶여 있는지 확인할 때

- **자연어 명령어 예시**:
  - "우리 dbt 프로젝트에 있는 모든 모델 목록을 보여줘"
  - "`stg_`로 시작하는 모델들만 골라서 보여줘"
  - "소스(source) 목록만 따로 알려줘"
  - "태그가 `nightly`인 모델들을 찾아줘"

---

### ② 특정 모델/소스 상세 정보 조회 (`get_node_details`)

- **무엇을 하나요?**
  특정 모델 하나를 골라 그 모델의 컬럼 목록, 각 컬럼의 데이터 타입, 설명(description), 설정된 테스트 등 세부 정보를 보여줍니다.
  마치 테이블의 "명세서"를 보는 것과 같습니다.

- **언제 쓰나요?**
  - "이 모델에 어떤 컬럼들이 있지?" 궁금할 때
  - 특정 컬럼이 어떤 테스트(제약 조건)가 걸려 있는지 확인할 때
  - 모델의 설정값(예: 매터리얼라이제이션 방식 - 테이블/뷰/인크리멘탈)을 확인할 때
  - 해당 모델의 원본 SQL 파일 경로를 알고 싶을 때

- **자연어 명령어 예시**:
  - "`orders` 모델의 컬럼 정보를 상세하게 보여줘"
  - "`customers` 모델에 어떤 테스트가 설정되어 있어?"
  - "`stg_payments` 소스의 설명과 컬럼 구조를 알려줘"

---

### ③ 데이터 리니지(흐름) 분석 (`get_lineage`)

- **무엇을 하나요?**
  특정 모델을 기준으로 데이터가 어디서 왔고(부모), 어디로 흘러가는지(자식)를 트리 구조로 보여줍니다.
  예를 들어, `final_report` 모델이 어떤 원본 테이블들에 의존하는지, 또는 `stg_orders`가 바뀌면 어떤 모델들이 영향을 받는지 알 수 있습니다.

- **언제 쓰나요?**
  - 데이터 오류가 났을 때 어디서 문제가 생겼는지 역추적할 때
  - 모델 하나를 수정하기 전에 "이걸 바꾸면 다운스트림(하위)에 얼마나 영향을 주지?" 확인할 때
  - 새로운 팀원에게 데이터 흐름 전체를 설명할 때

- **자연어 명령어 예시**:
  - "`final_sales` 모델의 데이터 흐름(리니지)을 설명해줘"
  - "`stg_users` 모델이 바뀌면 어떤 모델들이 영향을 받아?"
  - "`mart_revenue` 모델은 어떤 소스 데이터에서 왔어?"

## ⚙️ 기능 2: 프로젝트 검증 및 컴파일

### ④ 프로젝트 문법 검사 (`dbt parse`)

- **무엇을 하나요?**
  dbt 프로젝트 전체의 SQL 파일과 YAML 설정 파일을 훑어서 **문법 오류가 있는지 검사**합니다.
  실제 데이터베이스에 접속하거나 데이터를 바꾸지 않고, 코드 자체의 오류만 찾아줍니다.

- **언제 쓰나요?**
  - 새로운 모델을 추가하거나 기존 파일을 수정한 후 빠르게 문법 체크를 하고 싶을 때
  - 프로젝트가 정상적으로 인식되는지 확인할 때
  - 실행 전 기본적인 오류 여부를 빠르게 점검할 때

- **자연어 명령어 예시**:
  - "dbt 프로젝트에 문법 오류가 있는지 검사해줘"
  - "파싱(parse)을 실행해서 설정 파일에 문제가 없는지 확인해줘"

---

### ⑤ 모델 SQL 컴파일 (`dbt compile`)

- **무엇을 하나요?**
  dbt 모델 파일은 SQL과 함께 `{{ ref('other_model') }}`이나 `{{ var('my_variable') }}`같은 Jinja 템플릿 문법을 사용합니다.
  `compile`은 이 템플릿을 실제 데이터베이스에서 실행 가능한 **순수 SQL 코드로 변환**해주는 기능입니다.
  데이터베이스를 바꾸지 않고, 오직 "어떤 SQL이 실행될지" 미리 확인하는 용도입니다.

- **언제 쓰나요?**
  - "이 모델이 실제로 어떤 SQL로 실행되는지" 눈으로 확인하고 싶을 때
  - 복잡한 Jinja 로직이 의도대로 동작하는지 사전 점검할 때
  - 실행 전 쿼리를 검토하고 싶을 때

- **자연어 명령어 예시**:
  - "`weekly_report` 모델의 최종 SQL을 컴파일해서 보여줘"
  - "프로젝트 전체를 컴파일해서 오류가 있는지 확인해줘"
  - "`stg_orders` 모델이 실제로 어떤 SQL로 실행되는지 알려줘"

---

## 🚀 기능 3: 데이터 실행 및 빌드

### ⑥ 모델 실행 (`dbt run`)

- **무엇을 하나요?**
  선택한 모델(SQL 파일)을 실제 데이터베이스에서 **실행하여 테이블 또는 뷰를 생성/갱신**합니다.
  dbt의 핵심 기능으로, 가장 자주 사용하는 명령입니다.

- **언제 쓰나요?**
  - 데이터를 새로 만들거나 최신 데이터로 갱신하고 싶을 때
  - 특정 모델 하나만 다시 실행하고 싶을 때
  - 특정 태그나 이름 패턴으로 일부 모델만 선택해서 실행할 때

- **💡 초보자 팁**: 모델 실행 후에는 반드시 테스트(`dbt test`)를 함께 실행하는 것이 좋습니다. 아니면 아래의 `dbt build`를 사용하면 한 번에 처리됩니다.

- **자연어 명령어 예시**:
  - "지금 바로 dbt run을 실행해줘"
  - "`mart_`로 시작하는 모델들만 실행해줘"
  - "`orders` 모델과 그 하위 모델들을 전부 실행해줘"
  - "`stg_orders` 모델을 완전히 새로 다시 만들어줘 (full-refresh)"

---

### ⑦ 통합 빌드 (`dbt build`)

- **무엇을 하나요?**
  **모델 실행(`run`) + 테스트(`test`) + 시드 데이터 적재 + 스냅샷**을 dbt가 자동으로 순서를 결정하여 **한 번에 모두 실행**합니다.
  가장 안전하고 권장되는 실행 방식입니다.

- **`dbt run`과의 차이점**:
  - `dbt run`은 모델만 실행합니다 (테스트 없음)
  - `dbt build`는 실행 + 테스트를 한 번에 합니다. 중간에 테스트가 실패하면 더 이상 진행하지 않아 데이터 오염을 막습니다.

- **언제 쓰나요?**
  - 데이터 파이프라인 전체를 안전하게 한 번에 돌리고 싶을 때
  - 배포 또는 정기 실행 시 (테스트까지 포함한 완전한 실행이 필요할 때)

- **자연어 명령어 예시**:
  - "dbt build를 실행해서 전체 파이프라인을 돌려줘"
  - "`mart_orders` 모델을 build로 실행해줘 (테스트 포함)"
  - "특정 모델과 그 상위 의존 모델들을 모두 build해줘"

---

### ⑧ SQL 즉시 실행 및 데이터 미리보기 (`dbt show`)

- **무엇을 하나요?**
  임의의 SQL 쿼리를 데이터베이스에서 **직접 실행하고 결과를 바로 보여줍니다.**
  모델 파일을 만들지 않아도 되며, 데이터 내용을 빠르게 확인할 때 유용합니다.

- **언제 쓰나요?**
  - "이 테이블에 데이터가 실제로 어떻게 생겼는지" 샘플을 보고 싶을 때
  - 간단한 집계나 필터링 결과를 즉석에서 확인하고 싶을 때
  - 디버깅: 특정 값이 데이터베이스에 있는지 바로 조회하고 싶을 때

- **자연어 명령어 예시**:
  - "`orders` 테이블의 데이터 5건만 보여줘"
  - "최근 7일간 주문 건수를 SQL로 조회해줘"
  - "`customers` 테이블에서 이메일이 없는 고객 수를 알려줘"

## 🧪 기능 4: 데이터 품질 테스트

### ⑨ 데이터 테스트 실행 (`dbt test`)

- **무엇을 하나요?**
  YAML 파일에 정의된 **데이터 품질 규칙(테스트)**을 실행합니다.
  대표적인 테스트 종류:
  - **Unique**: 이 컬럼에 중복값이 없어야 한다
  - **Not Null**: 이 컬럼에 NULL(빈값)이 없어야 한다
  - **Accepted Values**: 이 컬럼의 값은 특정 목록 안에만 있어야 한다
  - **Relationships**: 다른 테이블과 외래키 관계가 맞아야 한다

- **언제 쓰나요?**
  - 데이터를 갱신한 후 품질에 이상이 없는지 검증할 때
  - 파이프라인에 문제가 생겼을 때 어디서 데이터가 깨졌는지 확인할 때
  - 정기적으로 데이터 신뢰성을 모니터링할 때

- **자연어 명령어 예시**:
  - "데이터 테스트를 전체 실행해서 문제가 있는지 확인해줘"
  - "`orders` 모델의 테스트만 실행해줘"
  - "테스트에 실패한 모델이 있으면 뭐가 문제인지 알려줘"

## 📚 기능 5: 문서화 및 공식 문서 검색

### ⑩ dbt 문서 사이트 생성 (`dbt docs`)

- **무엇을 하나요?**
  프로젝트의 모든 모델, 소스, 컬럼, 리니지를 시각적으로 볼 수 있는 **HTML 문서 웹사이트를 자동으로 생성**합니다.
  생성된 사이트에서 전체 데이터 카탈로그를 브라우저로 탐색할 수 있습니다.

- **언제 쓰나요?**
  - 팀원과 데이터 구조를 공유하고 싶을 때
  - 전체 데이터 카탈로그를 시각적으로 확인하고 싶을 때

- **자연어 명령어 예시**:
  - "dbt 문서 사이트를 생성해줘"
  - "docs generate를 실행해서 문서를 최신화해줘"

---

### ⑪ dbt 공식 문서 검색 (`search_product_docs` / `get_product_doc_pages`)

- **무엇을 하나요?**
  `docs.getdbt.com`의 공식 문서를 AI가 직접 검색하고 해당 내용을 가져와 설명해줍니다.
  dbt 기능에 대한 정확한 공식 레퍼런스를 즉시 찾아볼 수 있습니다.

- **언제 쓰나요?**
  - "인크리멘탈(incremental) 모델이 어떻게 작동하는지" 공식 설명이 필요할 때
  - 특정 dbt 기능의 정확한 문법이나 옵션을 알고 싶을 때

- **자연어 명령어 예시**:
  - "dbt의 incremental 모델에 대해 공식 문서를 찾아서 설명해줘"
  - "dbt snapshot이 뭔지 공식 문서 내용을 바탕으로 알려줘"
  - "dbt에서 태그(tag) 사용하는 방법을 공식 문서에서 찾아줘"

## 🔁 기능 6: 고급 기능

### ⑫ 이전 상태 복제 (`dbt clone`)

- **무엇을 하나요?**
  이전에 성공적으로 실행된 dbt 상태(프로덕션 상태 등)에서 모델을 현재 개발 환경에 **복사(클론)**합니다.
  데이터웨어하우스가 Zero-Copy Clone을 지원하는 경우 테이블 전체를 복사하지 않고 빠르게 복제합니다.

- **언제 쓰나요?**
  - 개발 환경에서 실제 프로덕션 데이터와 동일한 구조로 테스트하고 싶을 때
  - CI/CD 파이프라인에서 변경된 모델만 새로 실행하고, 나머지는 기존 상태를 빠르게 복제할 때

- **자연어 명령어 예시**:
  - "이전 프로덕션 상태에서 모델들을 현재 환경에 클론해줘"

## 💡 실전 활용 팁

### 초보자를 위한 권장 작업 순서

```
1. dbt parse    → 문법 오류 없는지 먼저 확인
2. dbt compile  → 실행될 SQL을 눈으로 검토
3. dbt build    → 실행 + 테스트를 한 번에 (가장 안전)
```

### 복합 명령 예시

- "모든 모델 목록을 보여주고, `sales`가 포함된 모델의 리니지를 분석해줘"
- "`orders` 모델을 컴파일해서 SQL 확인 후, 문제없으면 바로 build까지 실행해줘"
- "테스트 실패한 게 있으면 원인을 찾아서 어떻게 고치면 좋을지 알려줘"

### 문제 해결
